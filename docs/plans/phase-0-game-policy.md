# Phase 0 - 게임 정책 초안 (v0.1)

## 목적
- MVP에서 실제 플레이 가능한 정책을 먼저 고정한다.
- 서버/클라이언트가 동일한 규칙을 사용하도록 기준 문서를 만든다.

## 1. 게임 모드
- 모드: `No-Limit Texas Hold'em Cash Game`
- 화폐: `무료 칩` (현금성 가치 없음)
- 테이블 인원:
  - 최소 시작 인원: `2명`
  - 최대 인원: `9명`

## 2. 테이블 시작/진행 정책
- 핸드는 `활성 플레이어 >= 2명`일 때만 시작한다.
- 테이블이 1명만 남으면 다음 핸드를 시작하지 않고 대기 상태로 전환한다.
- 새로운 유저 입장은 핸드 중에도 허용하되, 참여는 다음 핸드부터 가능하다.

## 3. 좌석/버튼/블라인드 정책
- 버튼(Button)은 핸드마다 시계 방향으로 1칸 이동한다.
- 기본 규칙(3인 이상):
  - 버튼 왼쪽이 SB
  - SB 왼쪽이 BB
- 헤즈업(2인) 규칙:
  - 버튼이 SB를 낸다.
  - 버튼이 프리플랍에서 먼저 액션한다.
  - 플랍 이후에는 버튼이 나중에 액션한다.

## 4. 스택/블라인드 정책 (MVP)
- 기본 시작 스택: `100 BB`
- 기본 스테이크(초기 1개만 운영):
  - SB/BB = `10/20`
  - 시작 스택 = `2,000`
- 향후 확장:
  - 로비에서 다중 스테이크 테이블 제공
  - 테이블별 최소/최대 바이인 분리

## 5. 액션/타이머 정책
- 기본 액션 시간: `30초`
- 타임뱅크:
  - 플레이어별 초기 타임뱅크 잔액: `60초`
  - 기본 액션 시간(`30초`)이 모두 소진되면 타임뱅크를 자동 사용한다.
  - 사용한 만큼만 차감하며 남은 시간은 유지한다.
  - 예: 타임뱅크 60초 중 10초 사용 시 잔여 50초 유지
- 타임아웃 처리:
  - 체크 가능 시 `자동 체크`
  - 체크 불가 시 `자동 폴드`
- 올인 상태 플레이어는 이후 라운드에서 액션 대상에서 제외한다.

## 6. 입장/퇴장/재접속
- 퇴장:
  - 핸드 중 퇴장 요청 시 현재 핸드 종료 후 반영
- 연결 끊김:
  - 즉시 `일시 이탈` 상태
  - `60초` 내 재접속 시 자리/스택 유지
  - 재접속 시 플레이어별 타임뱅크 잔여 시간도 유지
  - 재접속 실패 시 타임아웃 규칙에 따라 자동 액션
- 장기 이탈:
  - 연속 3핸드 이상 블라인드 미지급 시 테이블에서 제거

## 7. 칩 정책
- 무료 칩 리필 쿨다운: `24시간` (초안)
- 음수 스택 불가, 모든 정산은 서버 단일 트랜잭션으로 처리한다.
- 레이크는 MVP에서 적용하지 않는다.

## 8. 공정성/악용 방지 (MVP 최소)
- 카드 셔플/딜/승패 판정은 서버에서만 수행한다.
- 클라이언트 액션은 아래 항목 검증 후 반영한다.
  - 현재 턴 여부
  - 허용 액션 여부
  - 베팅 금액 유효성 (min raise, all-in)
- 비정상 요청은 거절하고 보안 로그에 기록한다.

## 9. UI/UX 정책 (MVP)
- 플레이어는 항상 다음 정보를 확인 가능해야 한다.
  - 자신의 차례 여부
  - 콜 금액
  - 최소 레이즈 금액
  - 남은 액션 시간
- 재접속 시 현재 핸드 상태를 즉시 복원한다.
