# Phase 0 - PRD 초안 (v0.1)

## 1. 제품 개요
- 제품명: `CC Poker`
- 목적: 누구나 브라우저에서 빠르게 즐길 수 있는 무료 온라인 포커 플랫폼 제공
- 핵심 가치:
  - 공정한 판정(서버 권위형)
  - 빠르고 명확한 테이블 UX
  - 재접속 가능한 안정성

## 2. 타겟 사용자
- 1차:
  - 캐주얼 포커 유저(친구들과 가볍게 플레이)
  - 기존 포커 룰을 아는 유저
- 2차:
  - 관전자/스트리머 커뮤니티
  - 시즌 경쟁(랭크) 선호 유저

## 3. MVP 범위
- 2~9인 No-Limit Texas Hold'em 캐시 게임
- 계정 또는 게스트 기반 입장
- 로비 -> 테이블 입장 -> 플레이 -> 결과 확인 -> 재입장 흐름
- WebSocket 기반 실시간 동기화
- 재접속 복구(핸드 상태 스냅샷)

## 4. 제외 범위 (MVP)
- 현금 거래/출금/실물 보상
- 복잡한 토너먼트(MTT)
- 고급 소셜 기능(친구/클랜/메신저)

## 5. 핵심 유저 스토리
1. 유저는 닉네임으로 빠르게 입장해 즉시 플레이할 수 있다.
2. 유저는 자신의 턴에 가능한 액션과 금액을 명확히 확인할 수 있다.
3. 유저는 네트워크가 끊겨도 재접속 후 같은 상태로 복귀할 수 있다.
4. 유저는 핸드 종료 시 승패/정산 결과를 신뢰할 수 있다.

## 6. 기능 요구사항

## 6.1 로비
- 테이블 목록 조회(스테이크, 인원, 상태)
- 테이블 입장/퇴장
- 현재 접속자와 좌석 현황 표시

## 6.2 테이블
- 좌석/버튼/SB/BB 표시
- 커뮤니티 카드/팟/스택 표시
- 액션 패널:
  - fold/check/call/bet/raise/all-in
- 타이머/타임뱅크 표시:
  - 기본 액션 타이머 `30초`
  - 타임뱅크 `60초` 잔여 누적(사용 후 남은 시간 유지)

## 6.3 서버 게임 로직
- 턴 기반 액션 검증
- 사이드팟 포함 정산
- 타임아웃 자동 처리
- 테이블 상태 브로드캐스트

## 6.4 계정/세션
- 게스트 진입
- 기본 계정 생성/로그인(선택)
- 세션 만료/재연결 처리

## 7. 성공 지표 (MVP)
- 테이블 완주율(입장 후 핸드 5회 이상): `>= 60%`
- 재접속 복구 성공률: `>= 95%`
- 승패/칩 정산 치명 오류: `0건`
- 액션 반영 지연 p95: `120ms 이하`

## 8. 리스크 및 대응
- 리스크: 실시간 동기화 불일치
  - 대응: seq 기반 이벤트 적용 + snapshot 복구
- 리스크: 액션 검증 누락으로 공정성 저하
  - 대응: 서버 단일 검증 계층 + 통합 테스트
- 리스크: 초기 UI 복잡도로 이탈
  - 대응: 입력 중심 단순 UI 우선, 연출은 단계적 강화

## 9. 출시 전략
- 내부 알파 -> 클로즈드 베타 -> 공개 베타
- 베타 기간 중 주간 배포, 지표 기반 우선순위 조정
