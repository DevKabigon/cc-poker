# Phase 0 - 비기능 요구사항/NFR + SLO 초안 (v0.1)

## 목적
- MVP 품질 기준을 숫자로 정의해 개발/테스트/운영 기준을 통일한다.

## 1. 성능 목표

## 1.1 서버 API/실시간 지연
- REST API 응답 시간:
  - `p95 < 200ms`
  - `p99 < 500ms`
- WebSocket 액션 반영 지연(서버 수신 -> 브로드캐스트 완료):
  - `p95 < 120ms`
  - `p99 < 250ms`

## 1.2 클라이언트 체감 품질
- 액션 버튼 입력 후 로컬 피드백 표시: `100ms 이내`
- 재접속 후 스냅샷 복구: `2초 이내`

## 2. 안정성/가용성
- MVP 가용성 목표: `99.5%/월`
- 테이블 프로세스 장애 시 영향 범위:
  - 단일 테이블 단위로 격리
  - 다른 테이블 진행은 유지
- 무중단 배포는 Post-MVP 목표로 두고, MVP는 짧은 점검 시간 허용

## 3. 확장성 목표
- MVP 목표 동시성:
  - 동시 접속 유저(CCU): `300`
  - 동시 활성 테이블: `40`
- 단일 테이블 인원: `2~9`
- 수직 확장 우선, 필요 시 테이블 샤딩(프로세스 분리) 확장

## 4. 데이터 일관성/내구성
- 게임 상태는 서버 메모리 런타임 + 이벤트 로그로 복원 가능해야 한다.
- 핸드 결과/칩 정산은 원자적으로 처리한다.
- 장애 복구 목표:
  - RPO(데이터 유실 허용): `1분 이내`
  - RTO(복구 목표 시간): `15분 이내`

## 5. 보안 요구사항
- 인증:
  - MVP: 게스트 + 기본 계정 로그인
  - 인증 방식: `HttpOnly 세션 쿠키` 기반
  - 세션 만료/재발급 정책 필수
  - WebSocket 업그레이드 시 세션 쿠키 검증 필수
  - CSRF/크로스사이트 보호를 위해 `SameSite` + `Origin` 검증 적용
- 레이트 리밋:
  - 연결 시도, 액션 이벤트, 인증 API 각각 제한
- 입력 검증:
  - 금액, 액션 타입, 테이블 상태 불일치 요청 차단
- 감사 로그:
  - 보안 거절 이벤트 저장(최소 30일)

## 6. 관측 가능성(Observability)
- 필수 메트릭:
  - 활성 테이블 수
  - 활성 플레이어 수
  - 액션 처리 지연(p95/p99)
  - 재접속 성공률
  - 액션 거절률
- 필수 로그:
  - 테이블/핸드 단위 추적 ID
  - 에러 스택/입력 페이로드 마스킹
- 알람 기준 예시:
  - WebSocket 지연 p95가 5분 이상 150ms 초과
  - 재접속 실패율 10% 초과

## 7. 테스트 게이트
- 백엔드:
  - 게임 엔진 핵심 규칙 단위 테스트 필수
  - 사이드팟/올인/타임아웃 시나리오 통합 테스트 필수
- 프론트:
  - 테이블 진입/액션/쇼다운/재접속 E2E 필수
- 릴리즈 차단 조건:
  - 치명 버그(승패 오판정/칩 불일치) 1건 이상
  - SLO 회귀 확인 시 배포 보류

## 8. 운영 정책
- 배포 주기: 주 1~2회 정기 배포 + 긴급 핫픽스
- 롤백 기준:
  - 배포 후 15분 내 주요 지표 급락 시 즉시 롤백
- 장애 대응:
  - Incident 로그 기록
  - 사후 회고(원인/재발방지) 48시간 내 작성
